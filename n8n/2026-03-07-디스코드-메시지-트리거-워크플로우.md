### Context
Discord에서 특정 메시지를 수신하면, 이를 트리거로 활용하여 n8n 워크플로우를 자동화하는 방법을 설계했습니다. 이 프로세스는 메시지를 수신하고 이를 기반으로 TIL 형식 문서를 생성한 후, 해당 내용을 GitHub 저장소에 푸시하고 README.md를 업데이트합니다.

### Core
- Discord에서 메시지가 수신되면, n8n의 Discord 노드가 이를 감지하여 워크플로우를 시작합니다.
- 그 다음, 수신 데이터를 에이전트로 전달하여 TIL 형식의 문서를 생성합니다.
- GitHub API를 사용하여 생성된 문서를 지정된 저장소에 푸시합니다.
- 최종적으로 README.md 파일을 업데이트하여 새 문서에 대한 인덱스를 추가하고, 작업 완료 메시지를 Discord로 반환합니다.

```json
{
  "discordTrigger": {
    "event": "messageReceived",
    "channel": "some-channel-id"
  },
  "fileCreation": {
    "agent": "TILConverter",
    "format": "Markdown"
  },
  "githubPush": {
    "repository": "your-repo",
    "branch": "main",
    "filePath": "TIL-entries/new-til.md"
  },
  "readmeUpdate": {
    "filePath": "README.md",
    "updateMethod": "appendIndex"
  },
  "discordResponse": {
    "event": "messageSend",
    "content": "새로운 TIL이 업로드되었습니다."
  }
}
```

### Insight
n8n을 처음 사용하면서 두 가지 중요한 내용을 배울 수 있었습니다. 첫째, 각 노드의 입력(input)과 출력(output)을 정확히 이해하면 워크플로우를 설계하는 것이 상대적으로 용이하다는 점입니다. 둘째, 복잡한 자동화 작업을 손쉽게 구현할 수 있어 외부 트리거와 API를 효과적으로 활용할 수 있었습니다. 특히, 후츠릿에게 이 과정을 배워 매우 감사하게 생각합니다.
