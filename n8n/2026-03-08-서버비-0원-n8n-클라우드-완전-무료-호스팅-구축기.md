### Context
n8n을 로컬이 아닌 클라우드에서 24시간 운영하고 싶지만, 유료 서버비가 부담되는 상황에서 구글 클라우드 플랫폼(Google Cloud Platform, GCP)의 프리트라이얼(Always Free) 옵션을 활용하여 무료 인프라를 구축하고자 했습니다.

### Core
* **GCP e2-micro**: 구글 클라우드 프리트라이얼 인스턴스로 서버 자원을 확보하였습니다.
* **Docker와 docker-compose**: n8n 환경을 컨테이너로 격리하여 설치하고 관리 효율성을 높였습니다.
* **Ngrok**: 고정 IP나 복잡한 네트워크 설정 없이 Ngrok을 활용해 웹훅(Webhook) 경로를 안전하게 노출시켰습니다. 다음은 docker-compose.yml 설정 예시입니다:

```yaml
docker-compose:
  version: '3'
  services:
    n8n:
      image: n8nio/n8n
      ports:
        - "5678:5678"
      volumes:
        - /your-local-path:/home/node/.n8n
      environment:
        - N8N_BASIC_AUTH_ACTIVE=true
        - N8N_BASIC_AUTH_USER=user
        - N8N_BASIC_AUTH_PASSWORD=password
        - N8N_HOST=localhost
```

### Insight
유료 클라우드 서비스가 왜 비싼지 알게 되었습니다. 완전 무료로 설정하려니 굉장히 까다로웠습니다. 특히, 도메인 주소 설정 시 DuckDNS 도메인을 사용해 Let's Encrypt 인증서를 발급받으려 할 때 CAA 오류가 발생해 여러 시간이 소요되었습니다. 결국 Ngrok으로 문제를 해결하여 무료 조합을 완성할 수 있었습니다.
